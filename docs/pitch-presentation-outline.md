# 5-Minute Pitch Presentation Outline

## Slide 1 — Title

**Smart Campus Safety App**

A mobile application that helps students report campus safety incidents using GPS, camera and real-time data storage.

## Slide 2 — Problem

Students may notice hazards or unsafe situations on campus, but reporting is often slow, unclear or not location-aware.

Key issues:

- Incidents are not reported quickly.
- Location details may be inaccurate.
- Evidence is difficult to attach.
- Campus staff may not have structured incident records.

## Slide 3 — Solution

The Smart Campus Safety App allows students to:

- Register/login securely.
- Report incidents.
- Attach GPS location.
- Capture camera evidence.
- View reports on a map.
- Track incident status.

## Slide 4 — Key features demo

Show:

1. Login/register
2. Dashboard
3. Incident report form
4. Camera and GPS attachment
5. Map marker
6. Incident details/status

## Slide 5 — Technologies used

- React Native + Expo
- Node.js + Express
- Firebase Authentication
- Cloud Firestore
- Firebase Test Lab
- Expo Location, Camera, Notifications, Battery and Sensors

## Slide 6 — Testing and results

Testing completed:

- Mobile validation unit tests
- Backend health/API tests
- Manual end-to-end scenario
- Firebase Test Lab testing plan

Expected result:

- User can submit and view incident reports successfully.

## Slide 7 — Lessons learned

- Mobile apps need clear permission handling.
- Firebase makes authentication and database integration faster.
- Expo Go is useful for fast testing.
- Real-device testing is important for GPS/camera/sensors.

## Slide 8 — Future improvements

- Admin dashboard for security staff.
- Firebase Storage for photo uploads.
- Real push notifications.
- Role-based access control.
- Emergency contact calling.

## Slide 9 — Q&A preparation

**Q: Why Firebase?**
Firebase provides authentication, Firestore database and Test Lab support, matching the assessment requirements.

**Q: What device features are used?**
GPS, map, camera, notifications, battery, accelerometer and gyroscope.

**Q: What is the backend used for?**
The backend provides API endpoints for incident handling, audit logging and future admin integration.
